# TARA_DESIGN_SYSTEM.md

## Purpose

This document defines Tara's visual identity, design principles, and user experience standards.

Its purpose is to ensure consistency across prototypes, Flutter implementation, future web experiences, and dedicated hardware explorations.

This document should guide all future design decisions.

---

# 1. Brand Identity

## Brand Name

Tara

## Product Description

A conversational learning companion that helps learners explore, understand, reflect, and grow through natural conversation.

---

# 2. Brand Personality

Tara should feel:

* Friendly
* Calm
* Curious
* Trustworthy
* Encouraging
* Intelligent
* Patient
* Approachable

Tara should NOT feel:

* Corporate
* Formal
* Judgmental
* Robotic
* Academic
* Exam-focused
* Overly playful

---

# 3. Design Philosophy

Tara should feel like:

"A companion I can talk to."

not

"Educational software."

The design should prioritize:

* Simplicity
* Clarity
* Focus
* Calmness
* Conversation

Every screen should reduce friction between the learner and the conversation.

---

# 4. Core Design Principles

## Principle 1 - Conversation First

The primary action should always be conversation.

All secondary functionality should remain visually subordinate.

---

## Principle 2 - Minimal Cognitive Load

Avoid overwhelming users with:

* Menus
* Dashboards
* Statistics
* Complex navigation

Users should immediately understand what to do.

---

## Principle 3 - Encouraging, Not Evaluating

Tara should support learning.

Tara should not feel like a test, assessment, or grading system.

---

## Principle 4 - Mobile First

All experiences should be designed primarily for mobile devices.

Desktop and tablet experiences should adapt from the mobile experience.

---

# 5. Visual Style

## Design Direction

Modern Minimalism

Keywords:

* Clean
* Spacious
* Soft
* Premium
* Friendly

Avoid:

* Heavy gradients
* Excessive shadows
* Busy interfaces
* LMS-style dashboards

---

---

# 7. Typography

Preferred Font:

Inter

Fallback:

System Default

Characteristics:

* Modern
* Readable
* Neutral

Avoid decorative or playful fonts.

---

# 8. Iconography

Style:

* Simple
* Rounded
* Minimal

Primary Symbols:

* Microphone
* Sparkles
* Conversation
* Curiosity

Icons should communicate function immediately.

---

# 9. Primary Components

## Talk Button

The Talk Button is Tara's primary interaction element.

Characteristics:

* Large
* Circular
* Highly visible
* Easy to tap

The Talk Button should be the most visually prominent element on the screen.

---

## Status Indicator

Used for:

* Listening
* Thinking
* Speaking

Should provide clear feedback without overwhelming the learner.

---

## Conversation Bubble

Used for future conversational experiences.

Characteristics:

* Clean
* Spacious
* Easy to scan

---

# 10. Voice & Tone

Tara should communicate like a supportive companion.

Examples:

Good:

"That's an interesting question."

"Let's explore that together."

"Would you like another example?"

Avoid:

"Correct."

"Incorrect."

"Assessment complete."

"Score: 8/10"

---

# 11. Accessibility

Design should support:

* Children
* Teenagers
* Adults

Requirements:

* High contrast text
* Large touch targets
* Clear typography
* Simple navigation

---

# 12. MVP Screens

Sprint 01 MVP Screens:

1. Home Screen
2. Listening Screen
3. Thinking Screen
4. Speaking Screen
5. Conversation Screen

No additional screens should be added unless they directly support MVP validation.

---

# 13. Success Criteria

A learner should be able to open Tara and immediately understand:

* What Tara is
* What Tara can do
* How to start a conversation

without instruction or onboarding.
