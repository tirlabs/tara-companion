# TARA_MVP.md

## Objective

The purpose of the Tara MVP is to determine whether learners voluntarily engage with a conversational learning companion during moments of curiosity, confusion, revision, exploration, or discussion.

The MVP is intended to validate user behavior, not hardware design.

---

# Phase 1 – Conversational Companion (App-Based)

## Goal

Validate whether learners naturally interact with Tara.

## Scope

* Voice input
* Speech-to-text
* AI conversation
* Text-to-speech
* Conversation history
* Parent-controlled access

## Out of Scope

* Dedicated hardware
* Preparedness analysis
* Learning analytics
* Assessments
* Dashboards
* Personalized learning models
* Evidence extraction

## Success Criteria

* Learner voluntarily initiates conversations
* Learner returns to Tara repeatedly
* Learner uses Tara across multiple topics
* Parents observe genuine usefulness

---

# Phase 2 – Learning Companion

## Goal

Understand how Tara participates in learning activities.

## Additional Capabilities

* Revision conversations
* Topic discussions
* Follow-up questioning
* Explain-back interactions
* Project and hobby discussions

Examples:

* School subjects
* Games
* Stories
* Science
* Personal interests

## Success Criteria

* Learners use Tara for learning-related interactions
* Learners use Tara for curiosity-driven exploration
* Explain-back conversations occur naturally

---

# Phase 3 – Companion Intelligence Exploration

## Goal

Investigate whether learning conversations reveal useful patterns.

## Exploration Areas

* Conversation patterns
* Learning behaviors
* Reflection
* Explanation quality
* Questioning styles

This phase is exploratory and research-oriented.

No assumptions are made regarding preparedness or intelligence.

---

# Phase 4 – Dedicated Companion Device (Future)

## Goal

Evaluate whether a dedicated physical companion improves engagement.

Potential Features

* Always-available voice interaction
* Simplified learner experience
* Reduced dependence on mobile devices
* Reduced distractions

This phase will only be considered if earlier phases demonstrate meaningful learner engagement.

---

# MVP Principle

Tara should validate learner value before investing in dedicated hardware.

The first question is:

"Will learners naturally talk to Tara?"

Only after this question is answered should hardware-specific decisions be explored.
